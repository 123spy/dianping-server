# 去哪评后端

## 项目简介

去哪评后端是一个基于本地生活场景的点评与优惠券系统，支持用户找店、评分、评论、收藏、优惠券购买、订单流转以及后台管理。

## 技术栈

- Java 21
- Spring Boot 3
- Spring MVC
- MyBatis-Plus
- MySQL
- Redis
- RabbitMQ
- Spring AOP
- Knife4j / OpenAPI
- Lombok
- Hutool

## 已完成功能

### 用户模块

- 用户注册、登录、退出登录
- 获取当前登录用户
- 个人信息查看与修改
- 管理员用户管理

### 店铺模块

- 店铺分类列表
- 店铺分页查询
- 店铺详情查看
- 关键词搜索、分类筛选、城市筛选
- 热评榜单
- 管理员店铺增删改查

### 评论 / 评分 / 收藏模块

- 用户发布评论、删除自己的评论
- 用户提交评分、撤销自己的评分
- 店铺评分展示、评论列表展示
- 收藏店铺、取消收藏、查看我的收藏
- 管理员评论管理、评分管理

### 优惠券 / 订单模块

- 店铺优惠券查询
- 优惠券增删改查
- 创建优惠券订单
- 订单支付、取消、退款
- 我的订单、订单详情
- 我的优惠券、后台优惠券订单管理

### 抢购与性能优化

- Redis 缓存店铺分页列表与店铺详情
- Redis ZSet 维护热评榜单
- Redis + Lua 实现优惠券抢购预扣库存与一人一单校验
- RabbitMQ 异步落单，削峰处理高并发抢购请求

## 项目亮点

- 基于 Redis 对热点店铺分页接口进行缓存优化，高并发压测下显著降低平均响应时间并提升吞吐量。
- 对店铺详情、列表、榜单等热点读接口设计缓存与失效策略，提升系统读性能。
- 使用 Redis Lua 脚本将“校验库存、扣减库存、一人一单判断”合并为原子操作，避免高并发下超卖问题。
- 使用 RabbitMQ 将抢购后的订单创建异步化，实现业务解耦与请求削峰，降低接口同步耗时。
- 基于 MyBatis-Plus 快速完成通用 CRUD、分页查询与 VO 封装，提高开发效率。
- 提供用户端与后台管理端统一接口体系，覆盖用户、店铺、评论、评分、优惠券、订单等完整业务闭环。
