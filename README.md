# 去哪评后端

## 项目定位

基于本地生活场景实现的点评与优惠券系统后端，覆盖用户找店、评分评论、收藏、优惠券下单支付、后台管理，以及高并发抢购、缓存优化、异步削峰等核心能力。

## 技术栈

- Java 21
- Spring Boot 3
- Spring MVC
- MyBatis-Plus
- MySQL
- Redis
- RabbitMQ
- Spring AOP
- Knife4j / OpenAPI 3
- Lombok
- Hutool

## 核心功能

- 用户注册、登录、退出登录
- 获取当前登录用户
- 个人资料查看与修改
- 基于角色的管理员鉴权

- 店铺评分提交、评分撤销、评分展示
- 收藏店铺、取消收藏、我的收藏

### 优惠券与订单

### 后台管理

## 性能与并发优化

### Redis 缓存

- 对店铺分页列表接口做热点缓存，降低高并发查询下的数据库压力
- 对店铺详情接口做缓存，提升详情页读取性能
- 使用 Redis ZSet 维护热评榜单

### 高并发抢购

- 使用 Redis + Lua 实现优惠券抢购原子校验
- 将库存扣减、一人一单校验放入 Lua 脚本，避免并发下超卖
- 使用 RabbitMQ 将抢购后的订单落库异步化，降低请求同步耗时

### 消息可靠性

- 为抢购异步链路补充消息幂等处理
- 设计重试队列与死信队列，支持失败重试与异常留痕
- 通过死信消费者记录最终失败消息，便于人工排查与补偿

## 项目亮点

- 完成用户、店铺、评论、评分、收藏、优惠券、订单、后台管理的完整业务闭环，不是单纯 CRUD 演示项目。
- 基于 Redis 对热点分页列表接口进行缓存优化，结合压测显著降低平均响应时间并提升吞吐量。
- 使用 Redis Lua 将抢购场景中的库存校验、库存预扣、一人一单判断整合为原子操作，提升高并发正确性。
- 使用 RabbitMQ 对抢购后的订单创建进行异步削峰，并补充消息幂等、延迟重试、死信处理链路。
- 结合压测脚本验证缓存优化与抢购链路的效果，能够量化展示性能优化前后的响应时间与吞吐变化。
- 采用 MyBatis-Plus 统一实现分页、条件查询、VO 转换与后台管理接口，提升开发效率与代码一致性。

## 适合简历表达的项目描述

- 独立开发本地生活点评与优惠券系统后端，基于 Spring Boot、MyBatis-Plus、MySQL 完成用户、店铺、评论、评分、收藏、优惠券、订单及后台管理模块。
- 使用 Redis 对店铺分页列表、详情页、热评榜单进行缓存设计，并通过压测验证缓存优化效果，显著提升接口吞吐与响应速度。
- 针对优惠券抢购场景，基于 Redis + Lua 实现库存原子校验与一人一单控制，避免高并发下超卖问题。
- 基于 RabbitMQ 实现抢购后异步落单，补充消息幂等、延迟重试与死信队列机制，提升高并发链路的稳定性与可恢复性。

## 改进方案

短信验证码登录 / Redis 共享登录态
黑马点评的经典做法是短信登录，并把登录态放到 Redis 做共享 Session。这个仓库目前是账号密码登录，登录态走
request.getSession()；依赖里也能看到用了 Redis，但没有看到 spring-session 相关依赖。README 里还把“忘记密码、重置密码”列成了未完成事项。

附近商户 / GEO 搜索
黑马点评的一个代表功能是“附近的商户”，一般用 Redis GEO/GEOHash 做附近店铺查询。这个仓库的 Shop 查询虽然接收了
longitude、latitude，但实际查询条件主要还是 categoryId / city / name / address / tags
这些普通筛选，没有把经纬度用于距离检索；控制器里也看不到类似“nearby/near”接口。

用户签到（Bitmap）和 UV 统计（HyperLogLog）
这两个也是黑马点评课程里很有代表性的 Redis 题：签到通常用 Bitmap，UV 统计通常用 HyperLogLog。当前仓库的控制器、实体和 SQL
表里都没有看到对应的签到/UV 模块；资源目录下 Lua 也只有秒杀脚本。

关注体系 + 探店博客 + Feed 流
黑马点评不只是“店铺点评”，它还有一条比较核心的社交链路：关注、共同关注、探店博客、Feed 推送、点赞排行榜。这个仓库当前的后端模块主要是
comment / favorite / shopRating / couponOrder 这一套，没有看到 blog、follow、feed 相关控制器、实体或表。

“Redis 数据结构展示面”还不够全
黑马点评课程项目的特点，是会把 Redis 的 String / Hash / Set / Bitmap / HyperLogLog / GEO / List / ZSet / Lua
都串到业务里。这个仓库目前 Redis 主要集中在：店铺缓存、店铺评分榜 ZSet、秒杀 Lua 原子校验；但像收藏、评论这些模块，更多还是数据库直写，不像黑马点评那样把
Redis 各种结构充分铺开。
